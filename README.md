# pages-igame-cns

## 项目简介

本仓库用于归档与发布若干独立 HTML 页面。这些页面以静态形式存放，不依赖特定后端环境，可直接通过浏览器访问。仓库本身不针对任何单一站点或域名，仅作为页面集合的版本管理与分发用途。

## 目录结构

```
pages-igame-cns/
├── README.md              # 本文件
├── index.html             # 入口页面（可选）
├── pages/                 # 存放各独立 HTML 页面
│   ├── page-one/
│   ├── page-two/
│   └── ...
└── assets/                # 公共资源（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个子目录对应一个独立的 HTML 页面及其相关资源。
- `assets/`：存放多个页面可能共享的公共文件，按类型归类。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，不包含后端逻辑。
- 页面之间相互独立，互不依赖。
- 归档内容可能包含不同时期、不同用途的页面，不保证风格或主题一致。
- 页面可能引用外部资源（如 CDN 库），请确保网络环境允许加载。

## 维护说明

- 本仓库以归档为主要目的，不承诺定期更新。
- 提交新页面时，请保持目录结构清晰，避免覆盖已有文件。
- 如发现页面无法正常显示或链接失效，可提交 Issue 说明情况。
- 对于不再需要的页面，建议保留历史记录，避免直接删除。

## 使用方式

可通过以下方式浏览仓库中的页面：

1. 克隆仓库到本地，直接在浏览器中打开 HTML 文件。
2. 利用 GitHub Pages 等服务进行静态托管（需自行配置）。
